# Log In — API

## Description

Implement the login endpoint. Validate credentials against the stored investor record and issue a session token on success. Reject invalid credentials with an appropriate error response.

## Notes

- Endpoint: POST /auth/login
- Token strategy (JWT, session cookie, etc.) must be agreed with the team before frontend starts
- Invalid credentials must return a generic error — do not reveal whether the email or the password was wrong
