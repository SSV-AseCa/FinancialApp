# Sell Shares — API

## Description

Implement `POST /portfolio/transactions/sell` endpoint. Accepts: company CIK (or ticker), quantity (positive integer). Validates that the investor holds at least the requested quantity before executing. Atomically: creates a sell transaction record, decreases or removes the matching portfolio position, and updates the portfolio.

## Notes

- Returns 401 if not authenticated
- Returns 400 if quantity is not a positive integer or the company identifier is missing/invalid
- Returns 422 with a descriptive message if the investor does not hold sufficient shares
- The operation is atomic — partial state is never permitted
- An investor cannot sell shares they do not own — this must be enforced in the domain layer, not only in validation
- Depends on T04.02 API being in place (shared position/transaction model)
