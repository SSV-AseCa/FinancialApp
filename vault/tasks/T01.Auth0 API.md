# Auth0 Setup — API

## Description

Configure the Auth0 tenant and implement JWT validation middleware in the API. This is a prerequisite for every protected endpoint in cycles 2–5.

## Deliverables

- Auth0 tenant created with one Application (SPA or Native) and one API (resource server)
- API identifier registered in Auth0 (e.g. `https://api.yourdomain.com`)
- JWT validation middleware that:
  - Reads `Authorization: Bearer <token>` from every protected request
  - Validates the token signature against `https://your-tenant.auth0.com/.well-known/jwks.json`
  - Rejects expired or malformed tokens with 401
  - Extracts the `sub` claim and makes it available to downstream middleware
- All protected routes gated behind this middleware

## Notes

- This task must be completed before any other cycle 1 backend work on protected routes
- The Auth0 Application client ID and domain must be shared with Core, Web, and Mobile so they can configure the SDK
- Do not hardcode Auth0 credentials — use environment variables
- Lazy provisioning (T01.02 API) runs after this middleware in the chain
