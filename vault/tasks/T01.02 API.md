# Auto-Create Portfolio on Registration — API

## Description

Implement lazy provisioning middleware that runs on every authenticated request. On the first request from a given investor, create the investor record and an empty portfolio atomically. Subsequent requests skip provisioning.

## Notes

- Triggered by: any authenticated request where no investor row exists for the incoming `auth0_sub`
- Must be atomic — no investor without a portfolio, no orphan portfolios
- After provisioning, the middleware injects the internal `investor_id` into the request context for downstream handlers
- This middleware runs after JWT validation (see T01.Auth0 API) and before any business logic handler
- Enforces the invariant: exactly one portfolio per investor
