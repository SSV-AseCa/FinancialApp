# Register Account — Appium

## Description

Write the Appium E2E test for the mobile registration flow. Because Auth0 handles the authentication UI, tests must authenticate programmatically against Auth0's API rather than driving Auth0's screens.

## Scenarios

- Happy path: register via Auth0 → callback received → investor navigated to portfolio screen
- Post-registration access: authenticated investor can reach a protected screen

## Notes

- Obtain a test token directly from Auth0's token endpoint (Resource Owner Password grant or Management API) to bypass the Auth0 UI
- Duplicate registration and invalid credential errors are Auth0's responsibility — do not test Auth0's own UI
- Depends on T01.01.Mobile being complete
