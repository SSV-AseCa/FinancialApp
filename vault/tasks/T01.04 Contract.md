# Log Out — API Contract

## Description

Define and publish the API contract for the logout endpoint before any implementation begins. All other tasks in this story unblock once this is done.

## Contract Definition

- **Endpoint:** POST /auth/logout
- **Request:** how the current token is identified (header, cookie)
- **Success response:** status code, body (if any)
- **Post-logout behaviour:** document that any subsequent request with the invalidated token must return 401

## Notes

- This is a day-1 task — must be done at the start of the cycle
- Must be consistent with the token strategy defined in T01.03.Contract
- Publish alongside the login contract in the same document
