# Log In — API Contract

## Description

Define and publish the API contract for the login endpoint before any implementation begins. All other tasks in this story unblock once this is done.

## Contract Definition

- **Endpoint:** POST /auth/login
- **Request body:** credential fields, types
- **Success response:** status code, token format and location (body, header, cookie)
- **Error responses:** invalid credentials, missing fields — status codes and body shape for each
- **Token strategy:** document how the token is issued, where it is stored client-side, and how it must be sent in subsequent requests (Authorization header, cookie, etc.)

## Notes

- This is a day-1 task — it must be completed at the start of the cycle
- The token strategy decision affects T01.03.Core, T01.04.Core, and every authenticated request in future cycles — agree on it explicitly
- Publish as a shared document or OpenAPI spec
