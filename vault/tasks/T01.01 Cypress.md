# Register Account — Cypress

## Description

Write the Cypress E2E test for the web registration flow. Because Auth0 Universal Login is cross-origin, tests must authenticate programmatically via Auth0's Management API rather than interacting with Auth0's UI directly.

## Scenarios

- Happy path: register via Auth0 → callback received → investor redirected to portfolio screen
- Post-registration access: authenticated investor can reach a protected screen

## Notes

- Use `cy.loginByAuth0Api()` (Auth0's recommended Cypress command) to authenticate programmatically and bypass the cross-origin Universal Login page
- Duplicate registration and invalid credential errors are Auth0's responsibility — do not test Auth0's own UI
- Depends on T01.01.Web being complete
