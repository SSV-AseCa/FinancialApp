# Log In — Cypress

## Description

Write the Cypress E2E test for the web login flow. Because Auth0 Universal Login is cross-origin, tests authenticate programmatically via Auth0's API.

## Scenarios

- Happy path: programmatic Auth0 login → access token obtained → investor reaches portfolio screen
- Unauthenticated access: direct navigation to a protected route → redirected to login screen
- Post-login access: authenticated investor can reach protected screens

## Notes

- Implement `cy.loginByAuth0Api()` using Auth0's Resource Owner Password grant — this bypasses the cross-origin Universal Login page and is the approach Auth0 officially documents for Cypress
- Invalid credential errors are Auth0's responsibility — do not test Auth0's own UI
- Depends on T01.03.Web being complete
