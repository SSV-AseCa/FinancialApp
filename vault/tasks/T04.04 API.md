# View Transaction History — API

## Description

Implement `GET /portfolio/transactions` endpoint that returns all transactions for the authenticated investor in reverse chronological order. Each transaction includes at minimum: type (buy/sell), company identifier, quantity, and date.

## Notes

- Returns 401 if not authenticated
- An investor can only view their own transaction history — scope the query to the authenticated investor's ID
- Results must be ordered newest-first
- Depends on T04.02 API and T04.03 API (transaction records must exist before this endpoint is meaningful)
