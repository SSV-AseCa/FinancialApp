# Register Account — API Contract

## Description

Define and publish the API contract for the registration endpoint before any implementation begins. All other tasks in this story unblock once this is done.

## Contract Definition

- **Endpoint:** POST /auth/register
- **Request body:** fields, types, required vs optional
- **Success response:** status code, body shape
- **Error responses:** validation failure, duplicate email, server error — status codes and body shape for each

## Notes

- This is a day-1 task — it must be completed at the start of the cycle
- The contract is the output, not the implementation
- Publish it as a shared document or OpenAPI spec so all team members can start immediately
- Any change to the contract after publication must be communicated to the whole team
