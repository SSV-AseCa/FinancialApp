# Buy Shares — Appium

## Description

Write Appium E2E tests for the mobile buy-shares flow.

## Scenarios

- Happy path: investor selects a company, enters a valid quantity, buys, and the portfolio reflects the new/updated position
- Zero quantity: submitting zero quantity is rejected with a descriptive error
- Invalid company: submitting an unrecognized company identifier is rejected

## Notes

- Obtain a test token directly from Auth0's token endpoint
- Use a known stable company (e.g., AAPL) to keep tests deterministic
- Assert the portfolio position count or quantity increased after a successful buy
- Depends on T04.02 Mobile being complete
