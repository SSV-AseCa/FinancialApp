# Log In — Frontend Core

## Description

Expose a shared `login()` action that initiates the Auth0 login flow and stores the received access token on success. Consumed by both web and mobile screens.

## Notes

- Configure the Auth0 SDK with the tenant domain and client ID (provided by T01.Auth0 API)
- After the OAuth callback, store the access token securely — it must be included in the `Authorization: Bearer` header on all subsequent API requests
- Coordinate the action interface with Web and Mobile early so all three can work in parallel
- Keep the action decoupled from platform-specific navigation
