# Buy Shares — API

## Description

Implement `POST /portfolio/transactions/buy` endpoint. Accepts: company CIK (or ticker), quantity (positive integer). Atomically: creates a buy transaction record, creates or increases the matching portfolio position, and updates the portfolio. All mutations must succeed or none must be applied.

## Notes

- Returns 401 if not authenticated
- Returns 400 if quantity is not a positive integer or the company identifier is missing/invalid
- Returns 422 (Unprocessable Entity) for business-rule violations with a descriptive message
- The operation is atomic — partial state (transaction without position update, or vice versa) is never permitted
- Depends on T02.01 API (portfolio) and T06.01 API (company lookup) being in place
