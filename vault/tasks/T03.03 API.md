# Fetch and Store Company Financial Data — API

## Description

Implement a service that fetches financial metrics and recent SEC filings for a given company from EDGAR and persists them. This service is called on-demand when a company is first requested (e.g., after a search result is selected). All EDGAR requests must pass through the rate-limiting layer (T03.01 API).

## Notes

- Triggered by: a company being requested that has no stored financial data (or whose data is stale)
- Data to persist: key financial metrics (revenue, earnings, etc.) and a list of recent SEC filings (form type, filing date, URL)
- The EDGAR rate limiter (T03.01 API) must be in the call path — callers must not bypass it
- EDGAR API key is sourced from the `EDGAR_API_KEY` environment variable
- Stored data is consumed by S06.02 (View Company Financial Metrics), S06.03 (View Company SEC Filings), and S07.03 (View Watchlist Financial Metrics) in cycles 4–5
- Define and document the staleness threshold (how long stored data is considered fresh before re-fetching)
