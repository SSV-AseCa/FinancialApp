# Enforce EDGAR Rate Limit — API

## Description

Implement a rate-limiting layer at the EDGAR integration boundary that enforces a maximum of 10 requests per second. All code paths that call EDGAR must go through this layer — callers must not bypass it.

## Notes

- The limiter must be transparent to callers (they do not manage rate-limit state themselves)
- Must hold under concurrent request conditions, not just sequential ones
- Strategy choice (token bucket, sliding window, etc.) must be documented and justified
- This task is a prerequisite for S03.03 (Fetch and Store Company Financial Data) and all Company Research stories
